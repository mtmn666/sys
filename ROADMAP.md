# ロードマップ（Roadmap）

A → B → C の段階的拡張計画。

## フェーズ A：プロトタイプ（現在）

**目標**: Claude API を使った金利差分析が1回でも正しく動くこと。

- [x] SYSフォルダ構成の整備
- [x] 入力JSONスキーマの定義
- [x] Claudeへのプロンプト設計
- [ ] n8nで手動トリガーの最小フロー構築
- [ ] Claude APIへのHTTP Request設定
- [ ] 出力JSONの確認・ログ保存

**完了条件**: n8nから手動実行して、正しいJSON出力がレスポンスとして返ってくること。

---

## フェーズ B：自動化

**目標**: 毎朝9時に自動でデータ取得〜分析〜ログ保存が動くこと。

- [ ] 外部APIからUS10Y・JP10Y・USD/JPYを自動取得
- [ ] NewsAPI or RSSでニュース取得
- [ ] spread_yesterdayの自動引き継ぎ（前日ログから）
- [ ] スケジュールトリガー（毎朝9時）設定
- [ ] 実行ログの永続化

**完了条件**: 人が何もしなくても毎朝分析結果が生成されること。

---

## フェーズ C：拡張・出力強化

**目標**: 分析結果を外部サービスに自動投稿 + 変数を拡張。

- [ ] Notion自動投稿
- [ ] Slack通知
- [ ] VIX・2年債・株価の追加変数
- [ ] 過去データとの比較・トレンド分析
- [ ] ダッシュボード（Notion or カスタムUI）

**完了条件**: 毎朝Notionに自動でレポートが蓄積されること。

---

## 技術的な懸念点・メモ

| 懸念 | 対策 |
|---|---|
| ngrok URLが変わる | 有料プランへの移行 or Cloudflare Tunnel |
| APIレート制限 | リトライ処理をn8nに追加 |
| spread_yesterdayの欠損 | 初日はデフォルト値（直近の実値）を手動入力 |
| Claude出力がJSON以外になる | Function ノードでJSONパース失敗時のエラーハンドリング |
